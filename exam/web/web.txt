2. SCREENING ED ESCLUSIONE DELLE VULNERABILITÀ
Eseguo una serie di test preliminari sui campi di input per individuare la vulnerabilità presente ed escludere i vettori non sfruttabili.

2.1 Verifica Cross-Site Scripting (XSS)
- Test effettuato: Inietto nel campo di input la stringa <script>alert(1)</script>
- Esito: [OPZIONE A: La stringa viene stampata letteralmente a schermo o convertita in entità HTML, segno che l'input è sanitizzato. Escludo XSS] / [OPZIONE B: Il browser esegue il codice JavaScript mostrando il pop-up. La vulnerabilità è confermata].
[INSERIRE EVENTUALE SCREENSHOT TEST XSS]

2.2 Verifica SQL Injection (SQLi)
- Test effettuato: Inserisco un singolo apice ' o un doppio apice " nel campo di input per interrompere la query sul database.
- Esito: [OPZIONE A: La pagina restituisce un errore di sintassi SQL o cambia comportamento mostrando dati non previsti. La vulnerabilità è confermata] / [OPZIONE B: L'applicazione non mostra variazioni o filtra l'apice, segno che la query è protetta o parametrizzata. Escludo SQLi].
[INSERIRE EVENTUALE SCREENSHOT TEST SQLi]

2.3 Verifica Command Injection
- Test effettuato: Inserisco nel campo di input caratteri di concatenazione seguiti da un comando di sistema, ad esempio ; id oppure | ls
- Esito: [OPZIONE A: L'applicazione restituisce a schermo l'output del comando Linux, confermando l'esecuzione arbitraria di codice sul server] / [OPZIONE B: Il comando viene ignorato o trattato come testo semplice senza provocare l'esecuzione. Escludo Command Injection].
[INSERIRE EVENTUALE SCREENSHOT TEST COMMAND INJECTION]

2.4 Verifica Local/Remote File Inclusion (LFI/RFI)
- Test effettuato: Modifico il parametro della pagina inserendo un percorso di sistema come ../../../../etc/passwd o un URL esterno.
- Esito: [OPZIONE A: Il server legge e stampa a schermo il contenuto del file locale richiesto, confermando la falla di inclusione] / [OPZIONE B: L'applicazione restituisce un errore di file non trovato o non elabora il percorso. Escludo File Inclusion].
