## Fase 4: Regole Suricata (Mitigazione)
Sulla base delle evidenze estratte, è stata stilata la seguente configurazione per l'IDS/IPS.

### 4.1 (Opzionale) Configurazione variabili di rete
Nel caso in cui sia utile raggruppare le reti nel file `suricata.yaml`:
* `GOOD_NET`: "[INSERIRE_IP/SUBNET_LECITI]"
* `BAD_NET`: "!$GOOD_NET"

### 4.2 Regole Implementate
**(Scegliere/adattare la regola in base al tipo di attacco individuato)**

**Regola A: Blocco generico dell'host ostile (Ideale per Port Scan)**
```suricata
alert ip [IP_ATTACCANTE] any -> [RETI_VITTIMA] any (msg:"Rilevato traffico da host attaccante noto (Scansione/Intrusione)"; sid:1000001; rev:1;)
```
* *Spiegazione:* Avendo isolato l'IP con certezza, si allerta (o droppa) qualsiasi pacchetto IP (TCP, UDP, ICMP) proveniente dall'attaccante e diretto alle reti bersaglio.

**Regola B: Rilevamento specifico del Payload (Ideale per Buffer Overflow / Injection)**
```suricata
alert tcp any any -> [IP_BERSAGLIO] [PORTA_BERSAGLIO] (msg:"Rilevato tentativo di [TIPO ATTACCO]"; content:"[INSERIRE_IOC_ESTRATTO]"; sid:1000002; rev:1;)
```
* *Spiegazione:* La regola fa match specificamente sul payload `[INSERIRE_IOC]` diretto al servizio vulnerabile esposto dal server.
